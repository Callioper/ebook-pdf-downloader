# Dark Mode — Auto / Light / Dark Theme Support

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add dark mode with three theme options (auto/light/dark) selectable in settings, auto follows OS preference.

**Architecture:** Tailwind `darkMode: 'class'` strategy — a `dark` class on `<html>` triggers all `dark:` variants. A `useTheme` hook in Layout reads config `theme` field, applies the appropriate class, and listens for OS preference changes in auto mode. Theme preference is saved to backend config like all other settings.

**Tech Stack:** React 18, TypeScript, Tailwind CSS, Zustand store

```typescript
  // ── Theme ──
  const applyTheme = (theme: string) => {
    const root = document.documentElement
    if (theme === 'dark') {
      root.classList.add('dark')
    } else if (theme === 'light') {
      root.classList.remove('dark')
    } else {
      // auto: follow OS preference
      const mq = window.matchMedia('(prefers-color-scheme: dark)')
      if (mq.matches) { root.classList.add('dark') } else { root.classList.remove('dark') }
      const handler = (e: MediaQueryListEvent) => {
        if (e.matches) { root.classList.add('dark') } else { root.classList.remove('dark') }
      }
      mq.addEventListener('change', handler)
      return () => mq.removeEventListener('change', handler)
    }
  }

  useEffect(() => {
    fetch('/api/v1/config')
      .then(r => r.json())
      .then(cfg => {
        const theme = cfg.theme || 'auto'
        const cleanup = applyTheme(theme)
        return cleanup
      })
      .catch(() => {})
  }, [])
```
